EXECUTIVE SUMMARY

The Mutual Fund Analytics & Risk Intelligence Dashboard is an institutional-grade financial analytics platform developed to evaluate, compare, and visualize mutual fund performance using quantitative investment metrics. The project integrates robust data engineering, exploratory data analysis (EDA), financial performance analytics, risk modeling, and interactive dashboard development into a single end-to-end operational software workspace.

The system processes historical Net Asset Value (NAV) timelines, computes advanced risk-adjusted performance layers, generates automated fund rankings, and provides investors with actionable insights through a unified interactive user-terminal interface.

PROBLEM STATEMENT

Investors frequently struggle to perform thorough mutual fund due diligence due to fragmented information ecosystems spread across mismatched public ledgers. While basic trailing historical returns are commonly available, critical analytical vectors—such as downside risk metrics, alpha generation parameters, dynamic market sensitivities (beta tracking), and extreme tail-risk exposure statistics—are heavily siloed or hidden behind expensive data subscription paywalls.

This project directly bridges this operational data gap by providing a centralized analytics platform that combines automated backend extraction pipelines with comprehensive metrics calculations and front-end visualization engines.

DAY 4 – Fund Performance Analytics Engine

Objective: Apply quantitative finance algorithms to calculate risk-adjusted performance rankings.

Metrics Implemented:

Compound Annual Growth Rate (CAGR): Compiles smoothed compounding multi-period rate of historical return structures over an explicit 3-year trailing analysis horizon.

Sharpe Volatility Ratio: Evaluates risk-adjusted return efficiency earned per unit of total standard deviation, calculated against a customized 6.0% annualized risk-free asset floor.

Sortino Downside Ratio: Measures excess return generated relative strictly to downside semi-deviation, isolating harmful portfolio drops from upward growth movements.

Portfolio Alpha Generation: Tracks premium outperformance capabilities isolated entirely from broader index momentum characteristics.

Capital Asset Pricing Model (CAPM) Beta: Quantifies directional volatility and sensitivity profiles relative to baseline market index indicators.

Maximum Drawdown (MDD): Calculates peak-to-trough drop percentages to measure absolute structural downside asset vulnerabilities.

Composite Fund Score: A custom factor-weighted ranking algorithm that unifies multiple return-efficiency and loss-prevention markers into a single scale (1 to 100).

Outcome: Automated the production of a permanent performance scorecard registry (fund_scorecard.csv) for instant cross-sectional ranking and screening.

DAY 6 – Advanced Institutional Risk Intelligence

Objective: Integrate advanced mathematical layer workflows to support risk asset monitoring and tail-risk insulation analysis.

Advanced Metrics Coded:

Historical Value at Risk (Daily VaR 95%): Computes the lower 5th percentile return boundary to isolate maximum expected single-day capital losses at a 95% statistical confidence tier.

Conditional Value at Risk (Daily CVaR 95% / Expected Shortfall): Quantifies the expected average loss magnitude on catastrophic trading days when price drops break past the primary VaR boundary limit line.

Long-Term Lifetime Beta: Captures steady-state system baseline sensitivities over the asset's total operational lifespan.

Rolling 60-Day Beta Trends: Calculates dynamic, moving covariance matrices to track how fund managers adapt market exposure during shifting market conditions.

Operational Data Pipeline:

TECHNICAL CHALLENGES FACED & SOLUTIONS

Time-Series Continuity Discrepancies: Raw files had missing observation blocks on bank holidays and weekends.
Solution: Built a structured sequential forward-fill (ffill) and backward-fill (bfill) data transformation module to ensure uninterrupted time-series streams.

Dashboard Variable Reference Crashes: Streamlit threw a severe pathing lookup blocker ("NameError: name 'db_path' is not defined") if launched from differing directory trees.
Solution: Integrated Pathlib's "Path(os.getcwd())" to create an automated relative routing engine that anchors internal directory paths regardless of how the script is executed.

Query Latency Issues: Complex relational join operations created noticeable UI lag when changing fund select boxes.
Solution: Implemented storage caching decorators ("@st.cache_data") over the master SQL extraction scripts, compressing query runtimes down to milliseconds.
